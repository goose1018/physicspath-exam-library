## 🚨 修复历史（2026-04-29 跑 2023 全国甲卷时发现）

### Q1 真 bug（用户在浏览器实测发现，4 关 agent 漏抓）
- **现象**：铅球水平向前但**几乎不下落**（视觉只下移 ~1 像素）
- **根因**：`drawTrajectory(ctx,X,Y,w,h,state)` 函数参数 `h` 遮蔽全局 `const h=1.8`（推出高度 m）→ `pxPerM_y = ch/(h*1.05)` 中的 h 是面板像素高度（≈350px）而非物理 1.8m → 1.8m·pxPerM_y ≈ 1.4 px
- **修复**：函数参数重命名 `h` → `panelH`
- **教训**：4 关 agent 都基于代数推理 PASS，**抓不到 JS 变量遮蔽**这种语义陷阱
- **改进**：从 Q15 起 C/D 关 agent prompt 加「**变量作用域 / 遮蔽检查**」一项，明确扫全局 const vs 函数参数同名冲突

### Q12 真 bug（D 关 agent 抓到）
- **现象**：phases[0] 的 `\begin{aligned}...\end{aligned}` 多行公式在 q-shell.js（`displayMode:false`）下无法渲染
- **修复**：改成单行 `\quad` 间隔写法
- **教训**：q-shell.js 用行内模式 KaTeX，**不支持 aligned/align/cases/gather 等多行环境**
- **改进**：D 关 prompt 含「检查 formula_tex 是否含 `\begin{aligned}`」

### Q15 死代码 + 运算符优先级（D 关 agent 抓到）
- **死代码**：`seg1_t / total_t_jia / total_t_yi` 三个全局 const 从未被引用 → 已删
- **运算符优先级**：`progress >= 2 && |tNorm-1|<0.001 || progress === 2` 因 `&&` 优先于 `||` 等价于 `progress === 2`，前半冗余 → 已简化
